# Tournament Organizer（大会情報 共有ハブ）— モック

シングルエリミネーション大会の **情報を各所と共有する** ためのツールのモック。
運営作業そのものより、「本部が持つ1つの正（Single Source of Truth）を、
フロア / 配信 / ステージ進行 / クライアント等、必要な情報も連絡チャンネルも異なる
各ステークホルダーへ低コストで共有する」ことを主眼に置いている。

> 親リポジトリ（tcg-advance）とは独立した自己完結アプリ。Supabase は使わず、
> 状態は React Context + reducer、`localStorage` に永続化。バックエンド不要。

## 開発

```bash
cd tournament-organizer
npm install
npm run dev      # http://localhost:5174
npm run build    # tsc -b && vite build
```

## できること（モック）

- **セットアップ / インポート**: 大会作成、参加者をテキスト貼付（1行1名 or `名前,デッキ,所属` の CSV/TSV）または CSV/Excel ファイルから取り込み。並び順がシード順。最大32名。
- **ブラケット**: シングルエリミネーションを自動生成（2のべき乗でない人数は自動 BYE、3位決定戦オプション）。結果入力で勝者が自動的に次戦へ進む。事後の結果修正にも追従。
- **本部(Admin) ダッシュボード**: 試合カード一覧、進捗/進行中/待機の集計、ラウンド一斉開始、フィルタ（進行中 / 待機 / 配信卓 / 完了）。
- **試合ごとの管理**: タイマー（残り時間表示・延長 +3/+5/+10 分・停止）、卓番号割当、配信卓フラグ + メモ、結果入力、ペナルティ履歴（注意/警告/GL/ML/DQ・対象選手・メモ）。
- **アナウンス**: 本部が1回投稿 → 共有先（フロア/配信/ステージ/クライアント）を指定して各ビューに反映。固定（ピン）対応。
- **共有スナップショット**: 配信向け / ステージ進行向け / クライアント向けの要約テキストを生成し、クリップボードへコピー（Slack/LINE 貼付想定）。
- **会場モニター表示**: かっこいいブラケットの全画面表示（時計・進行中数・優勝者・お知らせティッカー）。
- **ロール切替**: 本部 / フロア / 配信 / ステージ進行 / クライアント。本部以外は派生の読み取りプレビュー（配信卓フラグや配信状況入力など、各ロールの「書き込み」スコープは今後追加）。

## モックの割り切り

- 認証なし。複数端末リアルタイム同期なし（共有ステートで概念を再現）。
- 実際の共有リンク配布の代わりに、1アプリ内のロール切替で各ビューを確認。
- 形式はシングルエリミネーションのみ（ダブルは未対応）。
