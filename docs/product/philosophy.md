**TCGマイスター**

**思想ドキュメント**

**プレイするのが、一番楽しい。**

Play is Fun.

プロジェクト情報源用ドラフト｜2026年5月

このドキュメントは、TCGマイスターが何を大切にするプロダクトなのかを整理するための思想文書です。機能一覧ではなく、なぜその機能が必要なのか、どのような体験を守りたいのか、何をやりすぎないべきなのかを言語化します。プロダクトの判断に迷った時は、まず「プレイするのが、一番楽しい。」という中心に戻ることを前提にします。

# **1. いちばん大事にしたいこと**

TCGマイスターが一番大事にしたいのは、シンプルに「TCGはプレイするのが一番楽しい」という感覚です。カードを集めることも、レアカードを引くことも、デッキを考えることも楽しい。けれど、カードゲームが本当に面白くなるのは、誰かと向き合って実際にプレイした時です。

手札を見て、相手の動きを読んで、迷って、勝負に出て、うまくいったり、負けたりする。その一戦の中に、自分の考えたことや、相手のうまさや、予想外の展開があります。最後に「いいゲームだった」と思える瞬間こそ、TCGの一番いいところです。

**プレイするのが、一番楽しい。**

# **2. カードの価値ではなく、プレイの楽しさを中心に置く**

TCGマイスターは、カードの価値を否定するサービスではありません。カードを集める楽しさも、欲しかったカードを手に入れる嬉しさも、TCGの大切な一部です。ただ、それでもやっぱり、カードゲームの中心にはプレイがあってほしい。

カードを持っていることだけではなく、そのカードを使って遊ぶこと、対戦すること、誰かと同じ時間を共有することに、もっと自然に目が向く状態を作りたい。強い言葉で誰かを否定するよりも、ただ「でも、プレイするのが一番楽しいよね」と言えるくらいの温度感が、TCGマイスターには合っています。

# **3. いま感じている違和感**

カードの周りには、カードを当てること、持つこと、価値が上がることに注目が集まりやすい空気があります。それ自体を否定する必要はありませんが、カードゲームなのだから、もっとプレイされていいはずです。

いい試合をした時の楽しさ、初めて会った人と気持ちよく対戦できた時の嬉しさ、またこの人と遊びたいと思える感覚は、TCGの本質的な魅力です。でも、その体験は今のままだとあまり残りません。カードの価値は記録されやすいのに、プレイの良さは消えてしまいやすい。そこに、少しもったいなさがあります。

# **4. TCGマイスターが作りたい空気**

TCGマイスターが作りたいのは、「もっとプレイしようよ」という空気です。ただし、それを押しつけたいわけではありません。プレイしない人を責めるのではなく、プレイした人が少し嬉しくなるようにしたい。良い対戦をした人、気持ちよく遊べる人、また一緒にやりたいと思われる人が、自然に見えるようにしたい。

アプリを使わせることが目的ではなく、アプリをきっかけに対戦が増えることが目的です。画面の中で完結するサービスではなく、実際のカードショップや大会会場や友人同士の対戦に、人がもう一度向かっていくためのサービスでありたいです。

# **5. 「良いプレイをするやつは、良いやつだ」**

TCGマイスターの中核にある価値観は、「良いプレイをするやつは、良いやつだ」という感覚です。ここで言う良いプレイは、単に強いプレイという意味ではありません。

相手をリスペクトしていること。勝っても負けても気持ちよく終われること。初対面でも安心して対戦できること。プレイ中の態度が良いこと。そういうものは、勝敗や順位とは別に、TCGの体験を支えている大事な価値です。

強いプレイヤーが評価されるのは当然です。でも、それだけではなく、「この人と対戦すると楽しい」「またこの人と遊びたい」と思われる人も、ちゃんと良いプレイヤーとして伝わるようにしたい。

# **6. プレイを記録する理由**

プレイは、その場では確かに存在しています。大会に参加したこと、誰かと対戦したこと、良いゲームがあったこと、対戦後に少し仲良くなったこと。でも何もしなければ、その多くはすぐに消えてしまいます。

TCGマイスターでは、その体験を重く記録したいわけではありません。細かいレポートを書かせたいわけでも、すべてを数値化したいわけでもありません。ただ、プレイした事実や、良い対戦があったことが、少しだけ残るようにしたい。

それによって、「この人とは前にも同じイベントにいた」「この人とはいい対戦をした」「この人はよくプレイしている」ということが自然に見えるようになります。そこから、また対戦する理由や、話しかけるきっかけが生まれます。

# **7. インセンティブの考え方**

プレイを増やすためには、プレイしたことに対して何らかのインセンティブがあってもいいと思います。ただし、そのインセンティブは慎重に設計する必要があります。

報酬のためだけにプレイする方向へ寄せすぎると、プレイそのものの楽しさが薄くなってしまいます。TCGマイスターが作りたいのは、プレイを作業にする仕組みではありません。

目指したいのは、プレイしたら少し嬉しい、良い対戦をしたら少し残る、また誰かと遊びたくなる、という循環です。インセンティブは、プレイの楽しさを置き換えるものではなく、プレイの楽しさを少しだけ増幅するものとして設計します。

# **8. 機能に落とすときの考え方**

TCGマイスターの機能は、すべて「プレイするのが一番楽しい」という思想に戻れるべきです。チェックインも、GGも、マナー評価も、履歴も、フォローも、それ自体が目的ではありません。すべては、プレイしたくなる気持ちを増やすため、良い対戦を増やすため、また遊びたい人とつながりやすくするためにあります。

## **8.1 チェックインの意味**

GPSチェックインは、単なる位置情報機能ではありません。「実際にその場に行って、プレイの場に参加した」ということを残すための仕組みです。TCGは、人が集まり、場所があり、対戦がある遊びです。だからこそ、その場にいたことには意味があります。

実際に参加した人だけが、そのイベントに紐づいた体験を残せる。これによって、記録や評価の信頼性が生まれます。チェックインは、プレイ体験の入口です。まず「そこにいた」ことを残す。その上で、誰と関わったのか、どんな良い対戦があったのかが少しずつ積み上がっていきます。

## **8.2 GGの意味**

GGは、「いいゲームだった」という気持ちを残すための機能です。レビューでも、採点でも、審査でもありません。対戦後に自然に生まれるポジティブな感覚を、軽く形にするものです。

ここで大事なのは、GGを重くしすぎないことです。相手を評価するというより、「楽しかった」「ありがとう」「またやりたい」に近いものとして扱いたい。勝敗とは別に残る、対戦後の余韻のようなものです。

## **8.3 マナー評価の考え方**

マナーや良いプレイの評価は、TCGマイスターの中でも特に慎重に扱うべき部分です。ここを強くしすぎると、ユーザー同士が監視し合うような場所になってしまいます。

目指したいのは、悪い人を見つけることではありません。良いプレイをした人が少し伝わることです。だから、設計としてはネガティブな通報や減点よりも、ポジティブな称賛を中心にします。「この人はちゃんとしている」「この人と対戦すると気持ちいい」「この人は初心者にも優しい」。そういう情報が、押しつけがましくなく自然に伝わる状態が理想です。

## **8.4 履歴の意味**

参加履歴や対戦履歴は、プレイヤーの活動の足あとです。ただし、それを競争ランキングのように見せすぎると、思想がズレてしまいます。大事なのは、「どれだけ偉いか」を示すことではなく、「どんなふうに遊んできたか」が分かることです。

どのイベントに参加したのか、どんな人と出会ったのか、どんなプレイの積み重ねがあるのか。それが見えることで、その人のTCGとの関わり方が自然に伝わります。これは実績解除というより、プレイの記憶に近いものです。

## **8.5 フォローや参加者一覧の意味**

フォローや参加者一覧は、対戦後の関係性を少しだけ続けるための機能です。TCGのイベントでは、いい対戦をしてもその場で終わってしまうことがあります。相手の名前も分からないまま別れて、二度と会わないこともあります。

でも、もし「あの人とまた遊べるかもしれない」と思えたら、TCGは少し楽しくなります。参加者一覧やフォローは、そのための橋です。知らない人にいきなり深くつながるのではなく、同じ場にいた人、対戦した人、また会いたい人と、ゆるくつながれるようにする。この距離感が大事です。

# **9. 思想と機能の対応**

機能は単なる便利機能ではなく、思想を現実の体験に変換するためのものです。それぞれの機能は、以下のような意味を持ちます。

| **機能** | **思想上の意味** | **守るべき温度感** |
|----|----|----|
| GPSチェックイン | 実際にプレイの場にいたことを残す | 証明ではあるが、押しつけない |
| GG | いいゲームだったという気持ちを残す | 採点ではなく、軽い称賛にする |
| マナー評価 | 気持ちよく遊べる人が伝わる | 監視ではなく、ポジティブな可視化にする |
| 参加履歴・対戦履歴 | プレイの足あとを残す | 競争よりも記憶に近づける |
| フォロー・参加者一覧 | 対戦後の関係性をゆるく続ける | SNS化しすぎず、プレイ起点にする |

# **10. やらないこと**

この思想を守るために、TCGマイスターはやりすぎないことも大切にします。便利だからといって、プレイの楽しさを損なう機能を増やしてはいけません。

- 人を点数で裁くような評価サービスにはしない。

- 悪い人を探すための監視サービスにはしない。

- 報酬のためだけにプレイさせる設計にはしない。

- 勝敗や順位だけが偉く見える設計にはしない。

- アプリ内の活動が目的化して、実際の対戦から遠ざかる設計にはしない。

# **11. 機能判断の基準**

新しい機能を検討する時は、次の問いに答えられるかを基準にします。この問いに対して明確に答えられない機能は、TCGマイスターの中心から外れている可能性があります。

- その機能は、プレイしたくなる気持ちを増やすか。

- その機能は、良い対戦を増やすか。

- その機能は、また遊びたい人とつながりやすくするか。

- その機能は、プレイの楽しさを邪魔していないか。

- その機能は、評価を重くしすぎていないか。

- その機能は、アプリではなくプレイを主役にしているか。

# **12. 一文で言うなら**

TCGマイスターは、カードを持つ楽しさではなく、カードで遊ぶ楽しさを中心に置くサービスです。TCGは、プレイするのが一番楽しい。だから、プレイしたこと、良い対戦をしたこと、また遊びたいと思える人に出会ったことが、少しだけ残るようにする。

勝敗だけではなく、良いプレイもちゃんと伝わるようにする。そうやって、もっと自然にプレイしたくなる空気を作っていきます。

**Play is Fun.**
