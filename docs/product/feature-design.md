# TCGマイスター収益化検討

*機能設計・O2O実装メモ｜情報源掲載用｜2026年5月*

結論：この文書は、TCGマイスターアドバンス側で実装すべき機能・データ取得・ユーザー体験・O2O特典設計を扱う。SSPや広告基盤の比較は別文書に分離し、本書では「広告商品化できる状態を作るためのプロダクト設計」に集中する。

| 区分 | 扱う内容 | 扱わない内容 |
|---|---|---|
| 本書の対象 | GPSチェックイン、QR認証、イベント後特典、クーポン、プッシュ通知、レシート投稿、ユーザー受容性、実装ロードマップ | SSP各社の比較、DSP接続、PMP/Deal IDの販売設計 |
| 根拠になる前提 | TCG Advance事業概要書の機能前提、TCGマイスター思想ドキュメントの体験方針、本セッションのO2O収益化議論 | 外部広告市場の最新比較調査 |
| 位置づけ | プロダクト・機能設計メモ | アドテック選定資料 |

## 1. 基本方針

TCGマイスターアドバンスは、報酬のためにプレイさせるサービスではない。中心に置くべき体験は「プレイしたことが少し残る」「良い対戦が少し嬉しくなる」「イベント参加後に近隣で使える特典が自然に出る」という状態である。事業側では、この体験を現地プレイ証明付きのローカルオーディエンスとして扱う。

## 2. Play Verified Local Audience

TCGマイスターが作るべき資産は、単なるアプリ広告枠ではなく、実際にTCGをプレイするために現地へ来たユーザーの検証済み行動データである。GPS、QR、イベント時間、会場ID、エリアID、アプリ内行動を組み合わせることで、通常の位置情報広告より文脈の濃いセグメントを作れる可能性がある。

| 要素 | 意味 | プロダクト上の実装 |
|---|---|---|
| Play | TCGを実際にプレイする目的で来ている | イベント参加、チェックイン、対戦後GG、参加履歴 |
| Verified | 現地参加が一定程度確認されている | GPS範囲判定、QR認証、時間窓、1イベント1回制限 |
| Local | 店舗・イベント会場・駅・商業施設と結びつく | 会場ID、エリアID、近隣特典、イベント後通知 |
| Audience | 外部企業にとって到達価値のある趣味コミュニティ | 匿名集計、広告/特典出し分け、クーポン反応計測 |

## 3. 実装すべき機能レイヤー

| レイヤー | 機能 | 目的 | 優先度 |
|---|---|---|---|
| プレイ証明 | GPSチェックイン、QR認証、イベント時間判定 | 実際に現地でプレイの場に参加したことを確認する | 最高 |
| イベント体験 | 参加者一覧、GG、フォロー、参加履歴 | プレイ体験を記録し、次の対戦や再来訪につなげる | 高 |
| O2O特典 | チェックイン特典、イベント後特典、近隣クーポン、利用ログ | プレイ後の自然な周辺消費を作る | 高 |
| 成果計測 | クーポン保存、利用QR、レシート投稿、表示/クリックログ | 広告主に説明できる反応・成果データを作る | 中〜高 |
| 通知・再来訪 | イベント前後プッシュ、再来店導線、期限付き特典 | イベント参加後の行動を促し、継続利用に接続する | 中 |

## 4. 取得・活用する変数

| 変数 | 取得方法 | 用途 | 根拠区分 |
|---|---|---|---|
| GPSチェックイン | 会場半径内で位置判定 | 現地来訪証明、特典表示条件 | 事実：既存機能前提 |
| QR認証 | 店舗/主催者のQRを読み取り | GPS補完、参加確度向上、不正対策 | 仮説：実装予定・有効性検証必要 |
| イベントフェーズ | イベント開始/終了時刻から判定 | 開催中・終了後特典の出し分け | 仮説：広告反応は未検証 |
| エリアID/会場ID | 店舗・会場マスタで管理 | 地域別・店舗別の商品設計 | 事実：設計可能 |
| リピート来訪 | チェックイン履歴から集計 | 習慣的来訪者の把握 | 仮説：広告価値は検証必要 |
| クーポン保存/利用 | アプリ内操作・QR利用 | 成果計測、広告主レポート | 事実：計測可能 |
| レシート投稿 | カメラ撮影・応募フォーム | 購買証明、キャンペーン成果 | 未確認：ユーザー受容性要検証 |

## 5. ユーザー向け表現

| 推奨表現 | 避ける表現 | 理由 |
|---|---|---|
| チェックイン特典 | 広告を見て報酬獲得 | プレイ目的を損なわず、自然な付加価値として見せられる |
| イベント参加者限定特典 | Earn、稼げる、ポイント稼ぎ | 報酬目的化を避けられる |
| プレイ後おすすめ | 動画視聴で特典獲得 | イベント後の消費タイミングに自然につながる |
| 近隣特典 | 広告枠、スポンサー広告 | ユーザー体験として違和感を減らせる |

## 6. 実装ロードマップ

| Phase | 実装内容 | 検証すること |
|---|---|---|
| 1 | GPSチェックイン、会場/エリアマスタ、イベント時間管理 | 現地参加データが安定して取れるか |
| 2 | QR認証、チェックイン完了面、イベント後画面 | GPS単独より信頼性が上がるか |
| 3 | チェックイン特典、近隣クーポン、保存/利用ログ | 特典導線がユーザーに受け入れられるか |
| 4 | プッシュ通知、イベント後30分導線、レシート投稿PoC | イベント後の反応率・利用率が高まるか |
| 5 | 広告/SSP連携用のKey-Value生成、セグメント管理 | 外部広告商品として扱える粒度に整理できるか |

## 7. やらないこと

| やらないこと | 理由 |
|---|---|
| 汎用バナー広告を主収益にしない | 単価が伸びにくく、ユーザー体験を毀損しやすい |
| 動画リワード/ポイント稼ぎ型に寄せない | 報酬のためにプレイする設計になり、思想とズレる |
| GGやマナー評価を広告変数として過度に売らない | コミュニティ品質の指標であり、広告価値は未検証 |
| 常時位置情報・過剰なトラッキングを初期から使わない | ユーザー信頼、審査、プライバシー説明の負荷が高い |

## 8. 未確認事項

| 未確認事項 | 検証方法 |
|---|---|
| イベント後特典のCTR/保存率/利用率 | A/Bテスト、イベント後画面のログ比較 |
| QR認証が広告価値に寄与するか | GPSのみ/QR認証ありで反応率を比較 |
| 近隣特典が広告臭く感じられないか | ユーザーインタビュー、継続率、離脱率分析 |
| レシート投稿が受け入れられるか | 応募型キャンペーンで小規模PoC |
| 外部企業が買う訴求になるか | 飲食・小売・飲料・決済・交通へのヒアリング |

最終結論：本書の対象は、広告基盤の選定ではなく、広告商品化できる現地プレイ証明とO2O導線をプロダクトに組み込むことである。SSP連携や広告販売は、この機能設計によって有効な1st party segmentが生成された後に検討する。
