**X to Earn・広告SSP連携検討**

*サービス類型・広告基盤・セグメント販売メモ｜情報源掲載用｜2026年5月*

結論：この文書は、X to Earn的な行動報酬モデルと、TCGマイスターアドバンスで取得される1st party segmentをどの広告基盤・SSPで扱い得るかを検討するための資料である。プロダクト機能設計は別文書「TCGマイスター収益化検討_機能設計メモ」に分離する。

| **区分** | **本書で扱う内容** | **本書で扱わない内容** |
|----|----|----|
| X to Earn類型 | Receipt / Move / Walk / Play / Check-in / Attention / Data to Earnなどの構造整理 | TCGマイスターの詳細UX設計 |
| 広告基盤 | SSP、GAM、PMP、Deal ID、1st party segment、Key-Value、SDA/PPSの考え方 | アプリ画面仕様、クーポンUI仕様 |
| 事業検証 | 広告主が買い得るセグメント、検証すべき単価・反応率・成果指標 | ショップ・主催者向け業務機能の詳細 |

# 1. X to Earn類型の整理

| **類型** | **ユーザー行動** | **主な収益源** | **TCGマイスターへの示唆** |
|----|----|----|----|
| Receipt to Earn | レシート撮影、購買証明 | 購買データ、販促、キャンペーン | レシート投稿は購買成果に近いが、初期は任意参加にする |
| Move / Walk to Earn | 歩行、移動、来訪 | 位置情報、健康、広告、自治体/スポンサー | カードショップへの移動・来訪はO2O価値になり得る |
| Check-in to Earn | 特定地点・イベントへのチェックイン | 送客、来店計測、地域回遊 | GPS/QRでイベント参加を検証できる点が中核 |
| Play to Earn | ゲームプレイ、対戦、継続参加 | 広告、ゲーム送客、課金、Web3 | TCGでは報酬目的化を避け、プレイ後特典に留める |
| Attention to Earn | 広告視聴、コンテンツ閲覧 | 広告費分配 | 動画リワード型は思想とズレるため主軸にしない |
| Data to Earn | 健康・行動・購買データ提供 | 分析、研究、広告 | 匿名集計・同意設計が重要 |

# 2. 広告市場で扱うべき変数

| **変数** | **広告市場での位置づけ** | **根拠区分** | **TCG側の扱い** |
|----|----|----|----|
| 現在地・近接 | ジオターゲティング、店舗周辺配信 | 事実：位置情報広告市場は存在 | area、venue_near、distance_bandとして扱う |
| 来訪/リピート | 来店履歴、ロイヤル層、再来訪 | 事実＋仮説：一般市場では価値あり、TCGでは検証必要 | repeat_venue_visitorとして扱う |
| QR認証 | 媒体固有の1st party segment | 事実：技術的に扱える。単価上昇は未確認 | qr_verifiedとしてKey-Value化 |
| イベント後時間帯 | 消費直前文脈の可能性 | 仮説：TCG文脈での反応は未検証 | event_post_30minとして扱う |
| クーポン保存/利用 | 広告反応・成果指標 | 事実：計測可能。広告価値は実測必要 | 自社レポート/セグメントに使う |
| GG/対戦後行動 | 高エンゲージメントの可能性 | 仮説：広告価値は未確認 | 初期は広告変数よりコミュニティ指標 |

# 3. 既存SSP・広告基盤候補

| **候補** | **扱える可能性があるもの** | **TCGマイスターでの使い方** | **未確認事項** |
|----|----|----|----|
| Google Ad Manager | Key-Value、広告ユニット、PMP/PG | qr_verified、event_phase、areaなどを広告リクエストに付与 | 小規模アプリでの導入条件、運用体制 |
| Google Publisher Provided Signals | 媒体1st party audience/context data | 標準タクソノミーに寄せられる文脈を買い手へ伝える | 使える分類粒度、利用条件 |
| PubMatic Audience Encore | Publisher 1st party dataのDeal ID販売 | Verified TCG PlayersのようなDeal化 | 初期規模での採用可否 |
| Magnite Access / DMP | Audience作成、セグメント化、商用化 | 規模が出た後のオーディエンス販売 | 日本での支援体制、アプリ面対応 |
| Equativ / SDA | Seller Defined Audiences / Contexts | ID依存を抑えたコホート型販売 | TCG独自分類の扱い方 |
| OpenX + Permutive | Publisher-owned consented dataのDeal販売 | 同意済み1st party segmentをPMP化 | 連携条件、最低規模 |
| Index Exchange / OpenRTB | Audience taxonomy / custom taxonomy | 買い手と合意した独自セグメント設計 | 直接導入可否より仕様理解が中心 |

# 4. SSPで売るべきDeal設計

| **Deal名イメージ** | **条件** | **想定広告主** | **根拠区分** |
|----|----|----|----|
| TCG Event Post-Play Audience | GPS/QR認証済み、イベント終了後30分以内 | ファミレス、コンビニ、カラオケ、交通 | 仮説：消費直前性の検証が必要 |
| Akihabara TCG Weekend Players | 秋葉原、週末、TCGイベント参加済み | 飲料、決済、ガジェット、求人 | 仮説：エリア×趣味×時間帯の価値検証が必要 |
| Verified Tournament Participants | 大会参加QR認証済み | エナジードリンク、軽食、通信 | 仮説：長時間滞在層としての価値検証が必要 |
| Repeat Local Players | 同一店舗/エリアへの複数回来訪 | 飲食チェーン、決済、ポイント経済圏 | 仮説：習慣行動としての価値検証が必要 |

# 5. Open AuctionではなくPMP/Deal IDを優先する理由

| **取引形態** | **TCGセグメントの伝わり方** | **判断** |
|----|----|----|
| Open Auction | 通常のアプリ広告枠として扱われやすく、QR/チェックインの文脈が価格に反映されにくい | 初期収益補助には使えるが本命ではない |
| Private Marketplace / Deal ID | 「TCGイベント参加者」「イベント後30分以内」などを説明した上で販売できる | 本命 |
| Programmatic Guaranteed | 特定広告主に固定条件・固定価格で販売しやすい | 実績後に検討 |
| 直販PoC | 広告主ヒアリングと成果検証を短期で行える | 初期検証に有効 |

# 6. 検証すべき指標

| **指標** | **目的** | **判断基準** |
|----|----|----|
| 通常広告枠とのCPM差 | 文脈付き広告枠が高く売れるかを見る | GAM上で広告ユニットを分けて比較 |
| CTR / 保存率 | イベント後特典への反応を見る | イベント後30分と通常画面を比較 |
| クーポン利用率 | 実際の送客価値を見る | GPSのみ/QR認証済みで比較 |
| レシート投稿率 | 購買証明まで到達するかを見る | 小規模キャンペーンで検証 |
| 広告主ヒアリング | セグメント価値が買われるかを見る | 飲食・小売・飲料・決済に提案 |

# 7. 最終整理

事実：位置情報や媒体1st party dataを広告取引に乗せる仕組みは存在する。未確認：TCGマイスターのQR認証済み・イベント後・リピート来訪セグメントが実際に高単価で買われるかは未検証である。仮説：TCGマイスターは、通常の位置情報広告では取りにくい「現地プレイ証明付きローカルオーディエンス」を作れるため、PMP/Deal ID型の商品化によって広告単価を高められる可能性がある。

次の打ち手：まずはプロダクト側でGPS/QR/イベント後/クーポン利用のログを取り、通常広告枠と高文脈広告枠を分けて反応率を比較する。その後、Google Ad ManagerのKey-ValueやPMP、さらにPubMatic/Magnite/Equativ等の1st party audience販売機能を検討する。
