# スイスドロー大会運営ツール（Swiss Ops）

トナメル / TCGマイスター / セカナビ のような大会運営の基本機能に加え、
**フロアスタッフ向け進行画面**と**オフライン入力**を備えた、運営側の大会運営ツールです。

プレイヤー向け LIFF アプリ（リポジトリ直下）とは**別アプリ**として独立しており、
このプロトタイプは**バックエンド不要・ローカル完結**で動作します。

## 起動

```bash
cd operations
npm install
npm run dev       # http://localhost:5180
```

その他のコマンド:

```bash
npm run build     # 型チェック + 本番ビルド
npm run test      # スイスエンジンの自己検証（組み合わせ・順位の不変条件）
npm run typecheck
```

## 主な機能

### 基本機能（運営ダッシュボード）
- **大会作成 / 設定**：BO1/BO3/BO5、制限時間、予定ラウンド数（自動算出: 参加人数の log₂ 切り上げ）
- **参加者管理**：個別追加・一括追加（1行1名、デッキ名付き）、ドロップ/復帰
- **スイスドロー組み合わせ**：
  - スコアグループ（マッチポイント）順の隣接ペアリング
  - **再戦回避**（バックトラッキングで再戦ゼロの完全マッチングを探索）
  - **Bye 自動付与**（奇数時、最下位かつ未Byeのプレイヤーへ。1人1回まで）
  - ドロップ者は組み合わせから除外
- **順位表 / タイブレーク**：MTG・ポケモンTCG 準拠
  - マッチポイント → OMW%（相手マッチ勝率）→ GW%（自ゲーム勝率）→ OGW%（相手ゲーム勝率）、各勝率の下限 33.3%

### 新機能①：フロアスタッフ画面
画面右上の「運営 / フロアスタッフ」切替で利用。タブレット等での運用を想定。
- **全卓の進行状況ボード**：未開始 / 対戦中 / 延長中 / 確定 をカード表示
- **制限時間タイマー**：開始時刻＋制限＋延長から残り時間を自動計算（残5分で警告色、超過で赤）
- **延長時間の付与**：+5分ボタンで卓ごとに延長、状態が「延長中」に
- **ペナルティ記録**：注意/警告/ゲームロス/マッチロス/失格/遅刻/デッキ登録ミス/遅延行為。
  マッチロス・失格は対戦結果へ自動反映
- **結果入力**：BO1 は勝者選択、BO3+ はゲームカウント入力（勝敗を自動判定）。誤入力の訂正も可能
- **共有表示**：入力内容は同じ大会を開いている運営・他スタッフ端末に**即時反映**

### 新機能②：オフライン対応
- 全状態は端末の **localStorage に永続化**。ネットワーク不調時もフロアスタッフ端末から結果入力でき、
  リロードしても保持されます（ローカルファースト設計）。
- オンライン/オフライン状態を画面右上に表示。

## アーキテクチャ

```
src/
├── engine/                  純粋ロジック（UI・永続化から独立。テスト対象）
│   ├── types.ts             ドメインモデル
│   ├── swiss.ts             スイス組み合わせ（再戦回避・Bye）
│   ├── standings.ts         順位・タイブレーク計算
│   └── selftest.ts          自己検証（npm run test）
├── store/                   状態・永続化・同期
│   ├── store.ts             localStorage + BroadcastChannel
│   ├── actions.ts           ドメイン操作（UIから呼ぶ唯一の窓口）
│   └── useStore.ts          React 購読フック（useSyncExternalStore）
├── components/              共通UI（MatchCard / ResultModal / PenaltyModal / ui）
└── pages/                   画面（Home / 運営各パネル / FloorStaffBoard）
```

### 「他の人も見れる」＆「オフライン入力」の実装方針

ローカル完結プロトタイプのため、複数端末間の共有は **`BroadcastChannel`**（同一オリジン・同一マシンの
複数タブ/ウィンドウ）で再現しています。`store.set()` が更新を全タブへ発行し、各画面が即時追従します。

将来 Supabase など実バックエンドへ拡張する際は、**`store/store.ts` の購読/発行と `store/actions.ts`
だけを差し替えれば**、エンジン・UI はそのまま流用できます（オフライン入力キュー → 復帰時同期も同層で実装）。

## 動作確認の手順（デモ）

1. `npm run dev` で開く
2. 大会を作成 →「参加者」タブで一括追加（8名以上推奨）
3. 「対戦表」タブ →「第1ラウンド開始」で組み合わせ生成
4. 右上を**フロアスタッフ**に切替 → 各卓で結果入力・延長・ペナルティを操作
5. **別タブで同じ URL を開く**と、片方の入力がもう片方へ即時反映される（共有表示の確認）
6. 「順位」タブでタイブレーク込みの順位を確認 → 対戦表で次ラウンド生成
